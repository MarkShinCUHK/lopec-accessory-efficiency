# Agents.md

## 개발 규칙

- 이 프로젝트는 로스트아크 경매장 악세서리를 현재 캐릭터 기준 로펙점수 상승량과 가격 대비 효율로 비교하는 웹 도구다.
- 기능 추가 시 계산 로직과 UI 로직을 분리한다.
- TypeScript를 기본으로 사용하고, API 응답 타입과 내부 도메인 타입을 명확히 나눈다.
- Lostark API JWT는 서버 환경 변수에만 두고 브라우저 번들에 포함하지 않는다.
- 후보 악세는 옵션만 따로 점수화하지 말고, 캐릭터 상태에 가상 장착한 뒤 전체 로펙점수를 다시 계산한다.
- 귀걸이/반지 2개 동시 교체 조합은 최종 장착 악세 세트가 같으면 중복 결과로 표시하지 않는다.
- 경매장 검색은 요청 제한을 고려해 대기, 재시도, 진행률 표시를 유지한다.
- LOPEC 수식 관련 변경 시 `npm run verify:lopec`로 최신 로펙 번들과 비교한다.
- 계산 로직 변경 후에는 최소 `npm run typecheck`를 통과시킨다.
- `.env`, `.cache`, `.next`, `node_modules`, `.playwright-mcp`, 스크린샷 같은 로컬 산출물은 커밋하지 않는다.
